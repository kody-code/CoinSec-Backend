# 个人记账软件数据库（PGSQL）

## 一、数据库总设计

- 数据库名：`coinsec_db`
- 核心表：7 张（v1.0 共 5 张，v2.0 新增 budgets、tags、record_tags）
- 设计原则：轻量、易维护、支持收支统计、分类管理、备注、时间筛选

## 二、数据表设计（5 张核心表）

1. 用户表（users）
   个人使用只需 1 条数据，用于登录、基础信息
   | 字段 | 类型 | 约束 | 说明 |
   |-------------|--------------|--------|--------------|
   | user_id | SERIAL | 主键 | 用户唯一标识 |
   | username | VARCHAR(50) | 非空 | 登录账号 |
   | password | VARCHAR(100) | 非空 | 加密密码 |
   | nickname | VARCHAR(50) | - | 昵称 |
   | avatar | VARCHAR(255) | - | 头像图片路径 / URL |
    | create_time | TIMESTAMP | 默认当前时间 | 账号创建时间 |
    | is_deleted | BOOLEAN | 默认 FALSE | 逻辑删除 |

2. 分类表（categories）
   管理收入 / 支出分类，如：餐饮、交通、工资、红包等
   | 字段 | 类型 | 约束 | 说明 |
   |-------------|--------------|------------|-----------------------|
   | category_id | SERIAL | 主键 | 分类 ID |
   | user_id | INT | 外键关联 users | 归属用户 |
   | name | VARCHAR(30) | 非空 | 分类名称 |
   | type | VARCHAR(10) | 非空 | income 收入 /expense 支出 |
   | icon | VARCHAR(255) | - | 分类图标地址 |
    | sort | INT | 默认 0 | 排序权重 |
    | is_deleted | BOOLEAN | 默认 FALSE | 逻辑删除 |

3. 账户表（accounts）
   管理资金来源：微信、支付宝、现金、银行卡等
   | 字段 | 类型 | 约束 | 说明 |
   |------------|---------------|------------|-----------|
   | account_id | SERIAL | 主键 | 账户 ID |
   | user_id | INT | 外键关联 users | 归属用户 |
   | name | VARCHAR(30) | 非空 | 账户名称 |
   | icon | VARCHAR(255) | - | 账户图标地址 |
   | balance | DECIMAL(12,2) | 默认 0.00 | 当前余额 |
    | status | INT | 默认 1 | 1 正常 0 停用 |
    | is_deleted | BOOLEAN | 默认 FALSE | 逻辑删除 |

4. 记账记录表（records）【核心主表】
   每一笔收入 / 支出都存在这里
   | 字段 | 类型 | 约束 | 说明 |
   |-------------|---------------|-----------------|-----------|
   | record_id | SERIAL | 主键 | 流水 ID |
   | user_id | INT | 外键关联 users | 归属用户 |
   | category_id | INT | 外键关联 categories | 所属分类 |
   | account_id | INT | 外键关联 accounts | 扣款 / 入账账户 |
    | type | VARCHAR(10) | 非空 | 收支类型 |
    | amount | DECIMAL(12,2) | 非空 | 交易金额 |
    | remark | VARCHAR(255) | - | 备注 |
    | record_time | TIMESTAMP | 非空，默认当前时间 | 记账时间 |
    | is_deleted | BOOLEAN | 默认 FALSE | 逻辑删除 |

5. 转账记录表（transfers）
    用于账户之间互转：微信 → 支付宝
    | 字段 | 类型 | 约束 | 说明 |
    |-----------------|---------------|---------------|-------|
    | transfer_id | BIGSERIAL | 主键 | 转账 ID |
    | user_id | BIGINT | 外键关联 users | 归属用户 |
    | from_account_id | BIGINT | 外键关联 accounts | 转出账户 |
    | to_account_id | BIGINT | 外键关联 accounts | 转入账户 |
    | amount | DECIMAL(12,2) | 非空 | 转账金额 |
    | transfer_time | TIMESTAMP | 非空 | 转账时间 |
    | remark | VARCHAR(255) | - | 备注 |
    | is_deleted | BOOLEAN | 默认 FALSE | 逻辑删除 |

6. 预算表（budgets）【v2.0 新增】
    用于设置月度/年度预算，跟踪各分类支出进度
    | 字段 | 类型 | 约束 | 说明 |
    |-----------------|---------------|---------------|-------|
    | budget_id | BIGSERIAL | 主键 | 预算 ID |
    | user_id | BIGINT | 外键关联 users | 归属用户 |
    | category_id | BIGINT | 外键关联 categories | 关联分类 |
    | budget_amount | DECIMAL(12,2) | 非空 | 预算金额 |
    | period_type | VARCHAR(20) | 非空 | 周期类型: MONTHLY/YEARLY |
    | period_year | INTEGER | 非空 | 年份 |
    | period_month | INTEGER | - | 月份（按月预算时填写） |
    | is_deleted | BOOLEAN | 默认 FALSE | 逻辑删除 |

7. 标签表（tags）【v2.0 新增】
    用于给记账记录打标签，多对多关联 records
    | 字段 | 类型 | 约束 | 说明 |
    |-----------|--------------|--------|-----------|
    | tag_id | BIGSERIAL | 主键 | 标签 ID |
    | user_id | BIGINT | 外键关联 users | 归属用户 |
    | name | VARCHAR(30) | 非空 | 标签名称 |
    | color | VARCHAR(20) | - | 标签颜色（HEX 色值） |
    | is_deleted | BOOLEAN | 默认 FALSE | 逻辑删除 |

8. 记录-标签关联表（record_tags）【v2.0 新增】
    用于实现记录与标签的多对多关系
    | 字段 | 类型 | 约束 | 说明 |
    |-------------|---------|------------|-----------|
    | record_id | BIGINT | 外键关联 records | 记录 ID |
    | tag_id | BIGINT | 外键关联 tags | 标签 ID |

## 三、数据库设计亮点

1. 结构极简：仅 7 张表，个人使用完全足够，无冗余
2. 支持完整场景
   - 收入 / 支出记账
   - 多账户管理
   - 分类统计
   - 转账记录
   - 时间筛选、金额统计
   - 预算管理【v2.0】
   - 标签系统【v2.0】
3. PostgreSQL 原生适配
   - 支持 DECIMAL 精确金额
   - 支持 TIMESTAMP 时间索引
   - 支持统计类 SQL（月账单、年账单）
4. 易扩展: 可后续加：报表、图片附件、周期性账单

## 四、业务逻辑说明

1. 用户：唯一操作者，管理自己的账单
2. 分类：区分收入 / 支出，用于统计开销结构
3. 账户：管理资金载体，记账时自动增减余额
4. 记录：核心流水，每笔消费 / 收入都生成一条记录
5. 转账：账户间资金转移，不影响总资产，只改变账户余额
6. 预算：按分类设置月度/年度预算金额，通过 overview 接口获取各分类已支出金额和进度百分比，前端自行决定是否提醒【v2.0】
7. 标签：自定义标签给记录分类，一条记录可关联多个标签，按标签筛选流水【v2.0】
